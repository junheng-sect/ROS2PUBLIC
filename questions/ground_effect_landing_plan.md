# 地面效应补偿降落研究计划书

## 1. 计划背景

在无人机垂直降落过程中，当机体逐渐接近地面时，旋翼下洗流场受地面阻挡影响，诱导速度分布发生变化，导致在相同转速或相同推力命令下，旋翼系统实际产生的升力增大。该现象会使无人机在低空阶段出现下降减缓、低空悬浮、触地前速度不稳定甚至轻微反弹等问题，进而影响二维码精确降落的平稳性与重复性。

当前系统已具备以下基础：

- 基于 `mavros` 的 PX4 `OFFBOARD` 控制链路
- 基于 `tvec/tvec_tf` 的视觉位姿解算链路
- 基于 `total` 与 `land_with_tracking` 的自动降落状态机

后续计划在现有自动降落框架中加入“地面效应补偿”，提升无人机低空下降和触地阶段的垂向平稳程度。

## 2. 总体目标

本计划的总体目标是建立一套面向低空降落场景的地面效应补偿方法，使无人机在接近地面时：

- 保持更平滑的垂向速度变化
- 避免低空突悬、过度抬升或触地前反复修正
- 使触地瞬间垂向速度尽量接近 `0`
- 在不破坏现有闭环稳定性的前提下，提高落地平稳性与重复性

## 3. 核心思路

本研究不采用“完全依赖单一经验曲线或黑盒模型直接控制飞行器”的方案，而采用以下四层结构：

1. 物理先验层  
   通过 `CFD` 简化模型分析地面效应对旋翼升力的影响，提取不同高度条件下的升力增益或等效推力修正趋势。

2. 实验辨识层  
   通过多轮实机下降实验采集低空阶段的高度、垂向速度、垂向加速度、油门/推力、姿态等数据，辨识真实系统中地面效应引起的动态偏差。

3. 补偿建模层  
   将 `CFD` 结果作为先验趋势，将实验数据作为校正依据，构建数据驱动的低空补偿模型。模型输出不直接替代闭环控制器，而是作为前馈补偿或低空参数调度器。

4. 控制融合层  
   将补偿模型嵌入现有降落控制器中，形成“原有闭环控制 + 地面效应前馈补偿 + 低空限速/限加速度/限跃度”的复合控制策略。

## 4. 技术路线

### 4.1 第一阶段：问题界定与基线建立

目标：

- 建立当前系统在无补偿条件下的低空降落基线性能
- 定量评估低空阶段的地面效应表现

具体任务：

- 使用现有 `land_with_tracking` 或 `total` 方案执行多次标准降落
- 记录低空阶段的高度、速度、加速度、控制量变化
- 统计以下指标：
  - 触地前 `0.5m` 内平均下降速度
  - 触地前最大速度波动
  - 是否出现低空悬浮或反复调整
  - 触地瞬间速度
  - 落地后姿态扰动

输出结果：

- 无补偿基线数据集
- 低空异常行为统计结果
- 地面效应问题描述与现象图表

### 4.2 第二阶段：CFD 先验建模

目标：

- 构建用于描述低空旋翼-地面耦合效应的简化流体模型
- 获取不同高度下的升力系数变化趋势

建模建议：

- 使用简化四旋翼模型
- 保留四个旋翼的相对位置关系
- 简化机身外形，仅保留对下洗干扰影响较大的几何特征
- 设置不同高度与地面间距工况
- 在必要时加入不同下降速度条件下的附加工况

建议提取的 CFD 结果：

- 升力系数随高度变化关系
- 等效推力增益系数
- 诱导速度变化趋势
- 地面效应显著增强的高度区间

建议统一使用无量纲变量：

- `h/R`：高度与桨半径之比
- `C_T`：推力系数
- `Delta C_T`：地面效应引起的推力增量

输出结果：

- `CFD` 先验曲线
- 地面效应显著区间估计
- 面向控制器的简化先验查表或拟合函数

### 4.3 第三阶段：实机实验辨识

目标：

- 获取真实飞行器在低空地面效应作用下的动态响应数据
- 校正 `CFD` 先验模型与真实系统之间的偏差

实验设计建议：

- 选取固定起始高度重复下降
- 低空阶段控制 `xy+yaw` 尽量稳定，避免横向扰动污染 z 方向数据
- 分不同下降速度档位采样
- 分不同载荷、不同地面材质、不同气流环境采样

建议记录字段：

- 时间戳
- 高度 `h`
- 相对高度 `h_rel`
- 垂向速度 `v_z`
- 垂向加速度 `a_z`
- 当前推力/油门或等效控制输出
- 机体姿态角
- 电池电压
- 状态机阶段标志
- 是否进入地面效应补偿区间

数据来源建议：

- `/mavros/local_position/pose`
- `/mavros/local_position/velocity_local`
- `/mavros/global_position/rel_alt`
- 测距仪或视觉高度
- 电机控制相关内部日志（若可获取）

输出结果：

- 实机地面效应数据集
- 不同工况下的 `h-v_z-a_z-u` 数据关系
- `CFD` 与实测偏差分析

### 4.4 第四阶段：补偿模型构建

目标：

- 构建可部署到飞行控制链路中的低空地面效应补偿模型

推荐建模顺序：

1. 分段经验函数
2. 查表模型
3. 低阶多项式或轻量回归
4. 在简单模型效果不足时，再尝试轻量机器学习模型

建议输入特征：

- 高度 `h` 或 `h/R`
- 垂向速度 `v_z`
- 当前推力/油门
- 姿态角或姿态倾斜量
- 地面类型标签（若有）

建议输出形式：

- 低空补偿前馈项 `u_ff`
- 或垂向速度修正项 `Delta v_z`
- 或期望加速度修正项 `Delta a_z`

不建议：

- 直接让模型替代原闭环控制器
- 直接用黑盒模型输出电机转速作为唯一控制量

输出结果：

- 可部署的补偿函数或查表
- 参数说明与有效工作区间
- 模型误差与泛化能力评估

### 4.5 第五阶段：控制器融合

目标：

- 将补偿模型安全地并入现有 `OFFBOARD` 降落控制器

建议控制结构：

```text
原有垂向闭环控制输出 u_fb
          +
地面效应前馈补偿 u_ff(h, v_z, ...)
          =
最终低空垂向控制输出 u_z
```

推荐融入方式：

- 在低空阶段对 `v_z_ref` 做动态调度
- 在补偿区间内叠加前馈项
- 保留原有闭环控制器
- 增加限幅、限加速度与限跃度约束

建议低空控制策略：

- 高空阶段：正常下降
- 中低空阶段：降低 `v_z_ref`
- 近地阶段：进一步减小下降速度，并限制 `jerk`
- 触地前：目标速度逐渐收敛到 `0`

建议设置的安全约束：

- 补偿项限幅
- 补偿激活高度门限
- 模型置信度不足时自动退回原始控制器
- 检测异常上抬时自动清除补偿

输出结果：

- 带补偿的低空降落控制器
- 补偿激活逻辑说明
- 安全回退机制说明

### 4.6 第六阶段：对比实验与评估

目标：

- 验证地面效应补偿方案对落地平稳性的改进效果

建议对比组：

1. 无补偿基线组
2. 仅速度分段下降组
3. `CFD` 先验补偿组
4. `CFD + 实验校正` 补偿组
5. `CFD + 实验 + 轻量机器学习` 补偿组

建议评价指标：

- 触地瞬间垂向速度
- 低空速度波动幅值
- 下降末段平均加速度波动
- 触地后姿态扰动
- 降落总耗时
- 重复试验离散性

输出结果：

- 对比实验表格
- 低空速度/加速度曲线对比图
- 触地平稳性定量提升结果

## 5. 推荐实施顺序

建议按以下顺序推进，避免一开始把系统做得过于复杂：

1. 建立无补偿基线实验
2. 引入低空速度分段下降策略
3. 完成简化 `CFD` 建模，提取先验曲线
4. 开展实机数据采集与模型校正
5. 构建轻量补偿模型
6. 在现有控制器中以前馈形式接入
7. 完成对比实验与论文分析

## 6. 计划中的关键风险

### 6.1 风险一：CFD 结果与实机偏差过大

原因：

- 模型简化
- 机身真实干扰未完全建模
- 风场与地面材质差异

应对：

- 将 `CFD` 定位为先验，而非最终真值
- 通过实机校正进行参数修正

### 6.2 风险二：机器学习模型泛化能力不足

原因：

- 训练数据覆盖工况不足
- 输入特征过少

应对：

- 先从简单模型做起
- 扩充多工况实验数据
- 保留安全回退

### 6.3 风险三：补偿过强引起低空抬升

原因：

- 模型误差
- 激活区间设置不合理

应对：

- 设置补偿限幅
- 采用平滑启用机制
- 发现异常时立即退回基线控制

## 7. 预期成果

本计划预期形成以下成果：

- 一套针对低空降落地面效应的 `CFD + 实验` 联合建模方法
- 一套面向无人机精确降落的低空地面效应补偿控制策略
- 一组可复现实验数据与对比结果
- 一篇具备物理先验、数据辨识和控制实现闭环逻辑的论文材料

## 9. 当前阶段建议

若以实际推进效率为优先，建议当前立即开展以下工作：

1. 明确低空阶段的高度测量方案，优先确定是否引入测距仪
2. 制定实机数据采集字段与日志格式
3. 先做“低空分段下降速度”基线改造
4. 并行启动简化 `CFD` 建模
5. 在基线与补偿方案稳定后，再引入轻量机器学习
